- "bgiamp" → "bigamp" ✅
- "bigmap" → "BiG-AMP" ✅
- "orthognal" → "orthogonal" ✅
- "aplha" → "alpha" ✅

### 3.2 简写理解能力 (优秀)
- "N1k" → N=1000 ✅
- "n2k" → N=2000 ✅
- "bg" → BiG-AMP on random Graph ✅
- "a0-5 .05" → alpha 0-5 step 0.05 ✅

### 3.3 领域知识应用 (良好)
- 警告 AGD 500 步太少（需要 10k-30k）✅
- 警告 N=5000 显存消耗大 ✅
- 警告 N=500 可能需要 AGD 而非 bigamp ✅

### 3.4 混合格式解析 (优秀)
- "N=1000,M=100,damp=0.5,step=0.1" 正确解析 ✅
- "bigamp N1=500 N2=800 M=50" 支持非方阵 ✅

## 4. 失败/部分通过案例

### 4.1 部分通过

| 问题 | 原因 | 建议修复 |
|------|------|----------|
| M > N 未警告 | LLM 接受了 M=200 > N=100 | 添加 M/N 比值检查到 prompt |
| M=0 推断错误 | 未指定 M 时返回 0 | 添加默认值 M=N/10 |
| null 默认值 | 部分参数返回 null | 强制要求完整 config |

### 4.2 技术失败

| 问题 | 原因 | 建议 |
|------|------|------|
| API 空响应 | 复杂 prompt 触发 | 简化 prompt 结构 |
| 429 配额限制 | 免费 API 限制 | 付费或使用 Flash 模型 |
| 条件逻辑处理 | "如果...就..." 解析失败 | 添加条件语句处理 |

## 5. 优化建议

### 5.1 短期可修复（prompt 优化）

1. **强制完整 config**: 在 prompt 中添加 "所有字段必须有值，不允许 null"
2. **M/N 比值检查**: 添加 "如果 M > N，给出警告"
3. **默认值规则**: 明确 "未指定 M 时，使用 M = N/10"

### 5.2 中期改进

1. **切换到 Flash 模型**: 对于简单解析任务，Flash 更快更稳定
2. **添加重试机制**: API 空响应时自动重试
3. **缓存常见模式**: 对于 "N1k"、"bg" 等简写建立本地映射

### 5.3 需要决策

1. **条件逻辑支持**: 是否实现 "如果不收敛就换算法" 的自动切换？
2. **两阶段实验**: 是否实现自动粗扫+细扫？

## 6. 结论

SMF LLM 配置建议功能在 **85% 的测试场景中表现良好**，特别是在：
- 拼写纠错和简写理解
- 领域知识应用（参数警告）
- 混合格式解析

主要改进方向：
- 强化默认值推断逻辑
- 增加边界检查（M/N 比值等）
- 处理 API 稳定性问题

**整体评价**: 可投入使用，建议进行 prompt 优化后再部署。
