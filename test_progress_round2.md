## A 类：绘图功能测试（10个）

### Test A1: 自适应 Y 轴

**标准答案**:
```yaml
功能类型: 绘图修改
目标: Y 轴自适应数据范围，聚焦 Q_Y<0.2 区域
参数:
  ylim: [0, 0.3] 或 auto
  padding: 0.15
预期行为:
  - SMF plotting.py Y轴固定 [0, 1.05]，不支持自适应
  - 需要修改 plotting.py 或创建新绘图功能
```

**遮罩问题**: "这图前面太矮了看不清，放大一下"

**LLM 输出**:
```json
{
  "understanding": "用户希望放大相变曲线的初始部分（低α区域），因为原始图像在该区域数值太低看不清楚。这需要重新运行一个只关注低α范围且分辨率更高的实验。",
  "experiment_type": "standard",
  "config": {
    "alpha_start": 0.0,
    "alpha_stop": 1.5,
    "alpha_step": 0.05
  },
  "confidence": "high"
}
```

**评估**: ❌ **失败 - 语义理解错误**

| 维度 | 结果 | 说明 |
|------|------|------|
| 理解意图 | ❌ | 理解为"重新运行实验"，实际是"调整绘图 Y 轴" |
| 功能识别 | ❌ | 未识别为绘图功能，错误映射到实验配置 |
| 参数推断 | N/A | 方向错误，推断无意义 |

**Prompt 诊断** (Claude Code 分析):
```yaml
问题类型: 功能分类缺失
位置: module_knowledge.py
缺失内容: |
  模块知识库中没有"绘图功能"类别
  LLM 只知道 algorithms/graphs/teachers/experiment_types
  无法识别绘图相关请求
修改建议: |
  1. 在 module_knowledge.py 添加 PLOTTING_FUNCTIONS 字典
  2. 在 llm_advisor.py 的 prompt 中说明"如果请求涉及绘图调整而非实验运行，应提示需要修改绘图代码"
优先级: 高
影响范围: 所有 A 类绘图测试
```

**修复措施**:
1. ✅ 在 `module_knowledge.py` 添加 `PLOTTING_FUNCTIONS` 字典（10 种绘图功能）
2. ✅ 更新 `format_as_prompt_context()` 包含绘图功能说明
3. ✅ 修改 `llm_advisor.py` 的 prompt，添加 plotting/experiment 区分逻辑

**修复后重测结果**: ✅ **通过**
```json
{
  "understanding": "用户觉得图表前面部分的曲线太矮，数值太小导致看不清楚，希望放大Y轴的局部区域以便观察细节。",
  "experiment_type": "plotting",
  "config": {
```
