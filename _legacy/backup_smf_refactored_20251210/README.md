# SMF - Sparse Matrix Factorization Framework

Teacher-Student 掩码矩阵分解实验框架，支持 GPU 加速和自然语言配置。

## 安装

```bash
cd /home/sucia/Sparse-Matrix
pip install -e .
```

## 快速开始

```bash
smf
```

直接输入自然语言描述实验需求：
```
N10000 M100, step 5000, 正交教师, alpha 0~4 间隔0.1, 画Q_Y和Q_Y_unobserved
```

## 命令

| 命令 | 说明 |
|------|------|
| `smf` | 启动实验向导（自然语言配置） |
| `smf -c FILE` | 从配置文件运行 |
| `smf --bg` | 后台运行 |
| `smf vis` | 浏览实验结果 |
| `smf log` | 查看后台运行日志 |
| `smf log -f` | 实时跟踪日志 |
| `smf resume` | 从检查点恢复 |
| `smf test` | 快速测试（默认参数） |

## 配置界面

启动后输入实验描述，AI 会解析并显示配置：

```
╭─────────────────────────────────────────────────────────────╮
│ AI Understanding: 使用正交教师进行10000×10000矩阵相变实验   │
╰─────────────────────────────────────────────────────────────╯
╭──────────────────────────────────────────────────────────────╮
│ [1]  算法    bigamp (5000 步)                                │
│ [2]  矩阵    10000×10000, M=100                              │
│ [3]  Alpha   0.0 ~ 4.0, 步长 0.1                             │
│ [4]  教师    orthogonal                                      │
│ [5]  指标    Q_Y, Q_Y_unobserved                             │
├──────────────────────────────────────────────────────────────┤
│      实验目标  研究正交教师的相变特性                         │
│      对比方向  Q_Y vs Q_Y_unobserved                         │
╰──────────────────────────────────────────────────────────────╯

修改序号 / clarify / y / n:
```

- 输入序号 `1-5` 修改对应参数
- 输入 `clarify` 追加描述
- 输入 `y` 确认开始
- 输入 `n` 取消

## 算法

| 算法 | 说明 | 推荐场景 |
|------|------|----------|
| `bigamp` | BiG-AMP 消息传递 | 大矩阵 (N>1000)，~200-5000 步 |
| `agd` | 交替梯度下降 | 小矩阵，~20k epochs |

## 教师模型

| 教师 | 说明 |
|------|------|
| `standard` | 标准高斯初始化 (默认) |
| `orthogonal` | 正交初始化 |
| `scaled_variance` | 缩放方差 |

## 指标

| 指标 | 范围 | 说明 |
|------|------|------|
| `Q_Y` | [0, 1] | 完整矩阵重合度 |
| `Q_Y_unobserved` | [0, 1] | 未观测位置重合度（泛化能力） |
| `Q_Y_observed` | [0, 1] | 已观测位置重合度 |
| `Q_W` | [-1, 1] | 左因子余弦重合度 |
| `Q_X` | [-1, 1] | 右因子余弦重合度 |

## 图结构

| 图 | 说明 |
|----|------|
| `random` | 随机图 (默认) |
| `dinic` | 双正则图 (Dinic 算法) |
| `low_loop` | 低循环图 |

## 后台运行

长时间实验建议后台运行：

```bash
smf --bg
```

或在配置确认后选择后台运行。

查看进度：
```bash
smf log -f
```

## 检查点恢复

实验意外中断后恢复：

```bash
smf resume
```

## 配置文件

也可以使用 YAML 配置文件：

```yaml
# config.yaml
matrix:
  N1: 10000
  N2: 10000
  M: 100

alpha:
  start: 0.0
  stop: 4.0
  step: 0.1

training:
  max_steps: 5000
  samples_per_alpha: 1
  seed: 42

algorithm_key: bigamp
graph_key: random
teacher_key: orthogonal
```

运行：
```bash
smf -c config.yaml
```

## 依赖

- Python >= 3.9
- PyTorch >= 2.0
- rich (进度条和 UI)
- numpy, scipy, matplotlib

GPU 加速需要 CUDA。

## 常见问题

**Q: 显存不足 (OOM)**
- 框架会自动检测显存并调整批量大小
- 对于超大矩阵 (N>20000)，会自动使用 FP16 存储

**Q: 进度条刷屏**
- 已修复，使用 Rich Live 固定位置更新

**Q: 如何查看 GPU 使用情况**
- 进度条第一行实时显示功率和显存
